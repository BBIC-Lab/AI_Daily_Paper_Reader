## 摘要
序列推荐系统通常通过交互历史的单次编码推断用户偏好，而不进行迭代细化，并依赖日益加深的架构来捕捉复杂模式。在这项工作中，我们从递归推断的视角重新审视序列推荐：用户偏好能否被建模为一个持久潜在状态，并递归地进行细化？我们提出RecRec（递归推荐），一个轻量级模型，它维护一个紧凑的潜在状态，并通过一个以交互证据为条件的共享递归模块进行更新。与以往的递归模型不同，RecRec引入了一种证据锚定校正机制，通过将每次更新锚定在原始交互上下文中来稳定细化过程，防止深度递归推理中的语义漂移。在标准评估协议下，三个基准数据集上的实验表明，RecRec在仅使用3.9M至14M参数的情况下，匹配或优于最先进的序列、基于图和推理增强型推荐器。消融研究表明，递归细化和证据锚定校正门都对性能有显著贡献，突显了递归潜在推断作为更深层次或基于语言的架构的可扩展替代方案的有效性。代码可在https://anonymous.4open.science/r/RecRec-6B67/README.md获取。

## Abstract
Sequential recommender systems typically infer user preferences through single-pass encoding of interaction histories without iterative refinement, relying on increasingly deep architectures to capture complex patterns. In this work, we revisit sequential recommendation from a recursive inference perspective: can user preferences be modeled as a persistent latent state that is recursively refined? We propose RecRec (Recursive Recommendation), a lightweight model that maintains a compact latent state and updates it through a shared recursive module conditioned on interaction evidence. Unlike prior recursive models, RecRec introduces an evidence-anchored correction mechanism that stabilizes refinement by grounding each update in the original interaction context, preventing semantic drift during deep recursive reasoning. Experiments on three benchmark datasets under standard evaluation protocols show that RecRec matches or outperforms state-of-the-art sequential, graph-based, and reasoning-enhanced recommenders while using only 3.9M to 14M parameters. Ablation studies demonstrate that both recursive refinement and the evidence-anchored correction gate contribute significantly to performance, highlighting the effectiveness of recursive latent inference as a scalable alternative to deeper or language-based architectures. Code is available at https://anonymous.4open.science/r/RecRec-6B67/README.md.

## 1. 论文的核心问题与整体含义

- **研究动机**  
  现有序列推荐系统（如 SASRec、BERT4Rec）通常对用户交互历史做一次前向编码来推断偏好，缺乏对偏好表征的迭代校正与逐步精化能力。模型只能靠加深网络来捕捉复杂行为模式，容易产生近似误差且无法恢复。

- **整体思路**  
  将序列推荐重新表述为一个递归潜在状态推断问题：用户偏好可被视作一个持久化的潜在状态，通过多次递归更新实现逐步细化。为此，提出轻量级模型 RecRec，在紧凑的参数规模下实现递归推理，同时引入“证据锚定”机制防止深层递归中常见的语义漂移（semantic drift）。

## 2. 方法论：核心思想与关键技术细节

- **核心架构**  
  RecRec 维护三种状态：  
  - 静态上下文 $x$：从交互历史的 frozen SBERT 嵌入经 mask 池化得到。  
  - 偏好状态 $y_t$：在第 $t$ 步对外输出的用户偏好表征。  
  - 潜在状态 $z_t$：内部推理用的记忆状态，用于支持递归更新。  

  初始时 $y_0 \leftarrow x$，$z_0 \leftarrow 0$。

- **递归细化与自校正**  
  在每个外步 $t\in\{0,\dots,T-1\}$ 内，对 $z$ 执行 $n$ 次内递归：  
  $$z^{(j)}_t = f_\phi([x \,\|\, y_t \,\|\, z^{(j-1)}_t])$$  
  其中 $f_\phi$ 为共享的非线性变换，$\|$ 表示拼接。  
  为防止语义漂移，引入**证据锚定校正门**：  
  $$g_t = \sigma(W_t[x \,\|\, y_t])$$  
  $$\tilde{z}_t = (1 - g_t)\,z^{(n)}_t + g_t\,x$$  
  该门值 $g_t$ 可视为一种数据依赖的信任系数，自适应地将更新后的 $z$ 与原始上下文 $x$ 插值，确保递归更新始终锚定于历史证据。

- **偏好状态更新**  
  利用校正后的潜在状态 $\tilde{z}_t$ 对 $y_t$ 做残差更新：  
  $$y_{t+1} = y_t + L \cdot \tanh\!\big(f_\phi([x \,\|\, y_t \,\|\, \tilde{z}_t])\big)$$  
  其中 $L$ 控制更新幅度。

- **深度监督与训练目标**  
  对每一步 $t+1$ 产生的偏好状态 $y_{t+1}$ 都计算交叉熵损失：  
  $$\mathcal{L}_{\text{CE}}(y_{t+1}, i_{\text{target}}) = -\log\frac{\exp(y_{t+1}\cdot e_{\text{target}}^\top / \tau)}{\sum_{j\in C}\exp(y_{t+1}\cdot e_j^\top / \tau)}$$  
  总损失为所有步损失的均值：  
  $$\mathcal{L}_{\text{total}} = \frac{1}{T}\sum_{t=0}^{T-1} \mathcal{L}_{\text{CE}}(y_{t+1}, i_{\text{target}})$$  
  梯度同时通过内递归和外更新反向传播，促使模型在每一步都产出逐渐更优的偏好表征。

## 3. 实验设计

- **数据集**  
  三个公开的 Amazon review 子集（来自[8]），涵盖不同领域和交互密度：  
  - Luxury Beauty（用户数 11,690，物品数 6,534，交互数 71,898）  
  - Video Games（用户数 64,039，物品数 33,611，交互数 508,508）  
  - Steam Games（用户数 67,610，物品数 10,978，交互数 5,023,170）  
  均过滤掉交互数少于 5 的用户和物品，并用 frozen SBERT 提取语义嵌入。

- **评估协议**  
  留一法（leave-one-out），每个测试样本与 99 个随机采样负例组成候选集（共 100 个物品）。采用 HR@1、HR@5、HR@10，NDCG@1、NDCG@5、NDCG@10 和 Prec@1、Prec@5、Prec@10 作为指标。

- **基准方法**  
  - 传统序列/图模型：LightGCN、NGCF、SASRec、UniRec、ReaRec。  
  - 大语言模型（LLM）推荐器：Allmrec（7B）、Tallrec（7B）、GPT4Rec（117M）。  
  与 LLM 对比时采用更小的历史长度和候选集（历史长度 10，候选集 20），遵循对应方法的原文设置。

## 6. 主要结论与发现

- RecRec 在仅 3.9M–14M 参数下，在 Luxury Beauty、Video Games、Steam Games 三个数据集上匹配或超越现有 SOTA 序列、图、推理增强及 LLM 推荐器。
- 递归细化与证据锚定校正门是性能的关键来源：移除校正门造成的性能下降最大，其次为移除潜在状态或内递归。
- 递归步数存在最优值（$T=7$ 附近），过深的递归会导致语义漂移和性能退化。
- 深度监督策略使模型在每一步都产生逐步优化的偏好表征，置信度分析显示早期递归能有效提升预测置信度。
- RecRec 的增益并不依赖于预训练的语义嵌入（SBERT），随机初始化也能获得类似性能，说明轻量递归框架本身具备强大的模式捕捉能力。

## 7. 优点（方法或实验设计上的亮点）

- **参数高效**：以极小的参数量（最低 3.9M）在多个数据集上取得有竞争力的结果，尤其比 LLM 方法少 99% 以上的参数，训练资源需求大幅降低。
- **递归推理范式新颖**：将推荐建模为持久化潜在状态的顺序精化过程，突破了单次编码的限制。
- **证据锚定校正机制**：通过可学习的校正门动态平衡递归更新与原始历史证据，有效防止语义漂移，是递归模型稳定性的关键创新。
- **深度监督训练**：对每个递归步的输出都施加监督，而非仅依赖最后一步，使模型每一步都具备可用的偏好表征，同时缓解深层递归的梯度问题。
- **充分的消融与鲁棒性分析**：不仅验证了各组件的作用，还通过随机初始化实验排除外部语义嵌入的干扰，证明了结构本身的优势。

## 8. 不足与局限

- **递归深度需手动设定**：最佳递归步数 $T$ 在不同场景下可能需要调参，论文未提供自适应的终止准则或理论上的收敛性保证。
- **误差累积仍存**：虽然校正门缓解了漂移，但随着递归步数超过最佳点，性能仍会下降，表明深层递归时的误差累积尚未完全解决。
- **领域和规模局限性**：仅在三个 Amazon 子集上评估，未扩展到更大规模、更稀疏或不同性质的推荐场景（如新闻、短视频流），泛化性有待进一步验证。
- **对比方法有限**：虽然覆盖了主流序列、图、递归及部分 LLM 推荐器，但未包含更近期的语言模型微调范式（如 InstructRec）、或基于扩散模型的推荐器等，对比的全面性仍可加强。
- **解释性缺失**：论文未讨论递归状态或校正门对人类可解释性的帮助，偏好细化的可解释路径未被探索。
- **对 frozen SBERT 的依赖**：实验主要使用 frozen SBERT embedding，虽然随机初始化也有相似性能，但文中并未探索其他初始化方式或端到端学习的可能性。
