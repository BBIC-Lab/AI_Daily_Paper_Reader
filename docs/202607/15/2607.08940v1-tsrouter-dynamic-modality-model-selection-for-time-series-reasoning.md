## 摘要
时间序列推理对现实问题解决至关重要。尽管大型语言模型（LLMs）和视觉-语言模型（VLMs）都能对时间序列数据进行推理，但它们的能力是互补的：LLMs 将时间序列处理为文本序列，从而保持精确的数值理解，但难以捕捉全局模式；而 VLMs 通过可视化时间序列高效捕获这些模式，但可能会丢失细粒度细节。此外，不同模型在任务特定专长和推理成本上差异显著。因此，为每个查询动态选择最合适的模态和模型至关重要，但这需要建模任务、查询、模态和模型之间的复杂交互，这些交互承载着丰富的上下文信号。为此，我们提出了 TSRouter，一个基于图的动态路由框架。TSRouter 构建了一个包含任务、查询、模态和模型节点的异构图，以语境化查询特征、模态属性和模型能力之间的交互。TSRouter 将路由形式化为候选评分问题，根据用户定义的性能-成本偏好评估每个模态-模型对，从而选择最佳候选。在 4 个不同时间序列推理任务上的综合评估表明，TSRouter 大幅优于各类基线，相对提升幅度达 16% 至 46%。此外，TSRouter 展现出对未见模型和新任务的鲁棒零样本即插即用泛化能力，并通过成本感知优化在保持高性能的同时降低计算开销。我们的代码可在 https://github.com/tianyi-lab/TSRouter 获取。

## Abstract
Time series reasoning is essential for real-world problem-solving. While both Large Language Models (LLMs) and Vision-Language Models (VLMs) can reason about time-series data, their capabilities are complementary: LLMs process time series as text sequences and thus preserve exact numerical understanding, but struggle with global patterns, whereas VLMs efficiently capture these patterns by visualizing time series but may lose fine-grained details. Moreover, models vary significantly in task-specific expertise and inference costs. Dynamically selecting the most suitable modality and model for each query is therefore crucial, yet challenging because it requires modeling the complex interactions among tasks, queries, modalities, and models, which carry rich contextual signals. To this end, we introduce TSRouter, a graph-based dynamic routing framework. TSRouter constructs a heterogeneous graph of task, query, modality, and model nodes to contextualize the interactions among query characteristics, modality attributes, and model capabilities. TSRouter formulates routing as a candidate scoring problem, where each modality-model pair is evaluated based on user-defined performance-cost preferences to select the optimal candidate. Comprehensive evaluations on 4 distinct time series reasoning tasks reveal that TSRouter substantially outperforms diverse baselines with 16\% to 46\% relative improvements. Furthermore, TSRouter demonstrates robust zero-shot plug-and-play generalization to unseen models and novel tasks and preserves high performance while reducing computational overhead through cost-aware optimization. Our code is available at https://github.com/tianyi-lab/TSRouter.

## 2. 方法论

### 核心思想
- 将路由问题转化为**候选评分问题**：给定查询 $q$，在候选集 $\mathcal{C}$（所有合法的模态‑模型对）中，按用户自定义的性能‑成本偏好选择得分最高的候选 $c^*$。
- 构建**异构图** $G=(V, E)$，包含任务、查询、模态、模型四类节点，以及任务‑查询、查询‑模态、模态‑模型、查询‑模型、查询‑查询（k‑近邻）五种边，用以刻画丰富的上下文交互。

### 关键技术细节
- **节点初始化**：通过预训练文本嵌入模型（如 Qwen3‑embedding）编码自然语言描述（任务要求、模型能力、模态属性、查询文本和时序统计摘要）作为节点特征，再经线性投影映射到共享隐空间。
- **图传播**：采用**异构图 Transformer（HGT）**进行多层消息传递，聚合类型感知的邻域信息以更新节点表示，并加入残差连接。第 $\ell$ 层的更新公式为：
  $$h^{(\ell+1)}_v = h^{(\ell)}_v + \text{HGTConv}\left(\{h^{(\ell)}_u : (u, r, v) \in E\}\right)$$
- **候选评分**：对于查询 $q$、模态 $m$、模型 $\ell$，先加性组合模态与模型表示，再与查询表示做元素乘，通过两层 MLP 得到分数：
  $$s(q, m, \ell) = \text{MLP}\left(h^{(L)}_q \odot (h^{(L)}_m + h^{(L)}_\ell)\right)$$
- **训练目标**：定义有效性分数 $e(q,c) = \alpha \cdot I(q,c) - (1-\alpha) \cdot \widetilde{\text{cost}}(q,c)$，其中 $I(q,c)$ 为回答正确指示函数，$\widetilde{\text{cost}}$ 为 min‑max 归一化成本，$\alpha$ 控制性能‑成本权衡。将真实有效性分数 softmax 后作为软目标分布，预测分数同样 softmax，训练目标为**KL 散度**：
  $$\mathcal{L} = D_{\text{KL}}\left(y_q \| \hat{y}_q\right)$$
- **零样本泛化**：新任务或新模型只需插入图节点并遵循相同边规则，无需额外训练即可进行路由。

### 算法流程
1. 收集训练数据：对每个查询遍历所有候选，记录正确性和成本。
2. 构建异构图并嵌入节点。
3. 多轮 HGT 传播得到最终节点表示。
4. MLP 预测候选分布，KL 散度损失优化。
5. 推理时直接选取得分最高的候选。

## 3. 实验设计

- **数据集**：主要使用 **TSRBench**，涵盖 4 类任务（感知、推理、预测、决策），共 15 个子任务，按 7:1:2 划分训练/验证/测试。另外用 **TSQA**（时间序列插补）和 **MTBench**（相关性预测）作为未见任务测试集，评估泛化能力。
- **基准方法**：
  - 规则型：始终选最大 LLM 或最大 VLM。
  - 学习型路由：**EloRouter**、**MFRouter**、**KNNRouter**、**Hybrid LLM**、**RouterDC**、**CausalLMRouter**、**GraphRouter**、**Router‑R1**。所有方法均将每个（模态，模型）对视为独立候选。
- **评估指标**：回答准确率（分类任务）、MSE 和 MAE（插补任务），以及 API 调用总成本（美元）。
- **实验配置**：使用 Qwen3‑8B/32B、LLaMA-3.3-70B-Turbo、Qwen3‑VL‑8B/32B、GLM‑4.5V 等作为候选模型；新增 Qwen3.5‑397B‑A17B 和 Kimi‑K2.5 进行泛化测试。

## 6. 主要结论与发现

- TSRouter 在 4 个时间序列推理任务上总体准确率达 **51.33%**，相对最佳基线提升 **16%∼46%**，且成本有竞争力。
- 方法具备**强零样本泛化**：能无缝集成新模型（准确率进一步提升至 53.5%），并有效适应全新任务类型。
- 通过调节 $\alpha$，TSRouter 可在几乎不损失成本效率的前提下**显著优于所有基线**，实现可控的性能‑成本平衡。
- 消融表明**异构图结构、查询间信息共享、MLP 评分头和模态‑模型解耦表示**都是性能关键。

## 7. 优点

- **联合模态‑模型路由**：首次将模态选择与模型选择统一在一个框架中，充分挖掘互补优势。
- **上下文感知的图建模**：用异构图显式捕捉任务、查询、模态、模型间的复杂交互，路由决策更精准。
- **即插即用泛化**：零样本适应新模型、新任务，无需重新训练，适合动态变化的模型生态。
- **成本可调控**：通过一个超参数 $\alpha$ 灵活权衡性能与开销，构建帕累托最优前沿。
- **轻量高效**：路由模型仅 576 MiB 内存，单次推理 3.21 ms，且支持批处理，便于实际部署。

## 8. 不足与局限

- **数据源单一**：主要评估基于 TSRBench，时序场景多样性有限；对真实世界极其复杂或高频时序的适用性待验证。
- **候选模型覆盖范围**：仅用了 8 个模型（4 LLM + 4 VLM），未涵盖更多架构（如编码器‑解码器、MoE 等）或领域专用模型，结论的普适性可能受限。
- **成本模型简化**：成本仅基于 API token 计价，未考虑不同硬件或本地部署的资源消耗差异。
- **依赖 LLM 生成节点描述**：虽然实验表明移除描述后性能下降不大，但初始特征的语义质量仍可能影响对完全陌生任务或模型的泛化。
- **训练数据获取代价**：需要为所有候选模型‑模态对收集每个查询的答案和成本，虽然只需要训练集，但若候选池扩大，前期开销会增长。
- **Query‑Query 边基于嵌入相似度**：可能引入噪声，且 k‑近邻的选取对结果有一定敏感性（见图 7c）。
