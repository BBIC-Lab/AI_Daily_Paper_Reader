## 摘要
预测建模是现代金融服务的核心组成部分，其中各类任务传统上使用基于人工设计的表格特征训练的独立模型来解决。这种针对特定任务的方法限制了复用，且难以充分利用异构数据源，如交易历史和数字交互信号。本文提出了一种基于多模态用户事件序列预训练基础Transformer模型的方法。来自多个数据源的事件被统一为单一的时间序列，从而能够早期融合异构模态，并通过下一个事件预测目标学习通用表示。这些表示与现有的人工设计用户特征相结合，在其上训练轻量级神经网络模型以完成多个下游任务。所提出的系统在减少开发开销的同时优于传统的特定任务模型。该方法已部署于东欧某大型银行的生产环境中，在业务指标上取得了可衡量的提升。

## Abstract
Predictive modeling is a core component of modern financial services, where a wide range of tasks are traditionally addressed using separate models trained on manually engineered tabular features. This task-specific approach limits reuse and makes it difficult to fully exploit heterogeneous data sources such as transaction histories and digital interaction signals. In this paper, we present an approach based on pretraining a foundation transformer model on multimodal sequences of user events. Events from multiple data sources are unified into a single chronological sequence, enabling early fusion of heterogeneous modalities and learning of general-purpose representations via a next-event prediction objective. These representations are combined with existing engineered user features, on top of which lightweight neural models are trained for multiple downstream tasks. The proposed system outperforms traditional task-specific models while reducing development overhead. The approach was deployed in production at one of the biggest banks in Eastern Europe, resulting in measurable improvements in business metrics.

### 2. 方法论核心
- **整体架构**：早期融合 + 预训练因果 Transformer + 冻结骨干 + 下游表格式神经网络（TabNN）。
- **事件序列融合与编码**：
  - 将交易（Txn）、点击流（Click）、通信（Comm）等多种模态的事件按时间戳**合并为单一序列**。
  - **嵌入层**：
    - 数值属性先 z-score 标准化 $\tilde{x}_{num} = (x_{num} - \mu_{num}) / \sigma_{num}$，再通过可学习线性变换投影到 $d$ 维：$\mathbf{a}_{num} = \mathbf{w}_{num} \cdot \tilde{x}_{num} + \mathbf{b}_{num}$。
    - 类别属性通过可学习嵌入表映射到 $d$ 维。
  - **事件编码器**：
    - 将同一事件的不同属性嵌入填充至固定维度 $N_{attr}$（缺失位置遮盖）。
    - 对属性的嵌入矩阵 $\mathbf{A}_e$ 应用**自注意力**（沿属性维度），捕获属性间关系，对补齐位置掩码。
    - 对注意力输出沿属性维做**掩码平均池化**，再通过 LayerNorm 得到单个事件表示 $\mathbf{e}$。
  - **Transformer 骨干**：采用 GPT‑2 风格的因果 Transformer，输入事件序列 $[\mathbf{e}_1, \dots, \mathbf{e}_T]$，输出上下文化的事件表示。用户表示 $\mathbf{z}_{seq}$ 通过时间维平均池化得到。
- **预训练目标**：根据时刻 $t$ 的隐状态 $\mathbf{h}_t$ 预测下一个事件 $t+1$ 的属性。
  - 对类别属性 $c$ 用线性+Softmax 预测类别概率，损失为交叉熵。
  - 对数值属性 $n$ 用线性层输出标量，损失为 MSE。
  - 每个位置 $t$ 仅对下一个事件真实出现的属性计算损失，总损失为：
    $$\mathcal{L}_t = \sum_{c\in\mathcal{C}_{t+1}}\!\!\mathrm{CE}(\mathbf{y}^{(c)}_{t+1},\hat{\mathbf{p}}^{(c)}_{t+1}) + \!\!\sum_{n\in\mathcal{N}_{t+1}}\!\!\mathrm{MSE}(y^{(n)}_{t+1},\hat{y}^{(n)}_{t+1})$$
- **下游模型**：
  - **表格式模型 TabNN**：对数值特征先做带符号的 log 变换再标准化，数值特征以 $\mathbf{w}_{j,num} \cdot \tilde{x}_j$ 方式嵌入；类别特征查表嵌入。所有特征嵌入拼接后加噪声，送入 Transformer 编码器（沿特征维度交互），最后用可学习权重加权求和得到 $\mathbf{z}_{tab}$。
  - **融合与训练**：冻结的序列编码器输出 $\mathbf{z}_{seq}$ 经 MLP 适配器得到 $\mathbf{z}'_{seq}$，与 $\mathbf{z}_{tab}$ 拼接，接任务特定的 MLP 头。训练时**仅更新适配器、TabNN 和任务头**，序列编码器保持冻结。
- **设计选择**：
  - 预训练目标经对比优于 CoLES、MLM 和 DeTPP。
  - TabNN 在内部数据集上优于 DCNv2、TabM 等架构，结构与 FT‑Transformer 相近但激活与归一化顺序不同。

- **线上价值**：A/B 测试总 NPV 提升约 **+1%**，在系统规模下带来显著绝对收益，已作为默认评分模型部署。

### 7. 优点
- **统一多模态早期融合**：摒弃模态独立建模，让 Transformer 直接捕捉跨模态时序交互。
- **生产友好**：冻结骨干复用，下游训练快速，显著降低开发与维护成本。
- **充分利用已有特征**：不替换现有业务特征，而是将其与序列嵌入互补，实用性强。
- **实验扎实**：包含强生产基线、多维度消融、线上验证，结果可信。
