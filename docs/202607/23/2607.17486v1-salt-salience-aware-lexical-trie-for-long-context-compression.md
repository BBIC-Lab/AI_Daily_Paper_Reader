## 摘要
随着大型语言模型（LLMs）处理越来越长的提示，计算和KV缓存内存成本已成为推理系统中的主要瓶颈。现有的输入级提示压缩方法解决了这个问题，但它们通过标量相关性评分对每个句子进行排序，将文档视为无结构的单词和句子集合。在预算紧张的情况下，这会导致主题塌缩，即文档的主导主题消耗了预算，丢弃了出现频率较低但对任务相关的主题。而保持主题覆盖范围则需要将预算分配到各个重复出现的主题上，而不是孤立地对句子打分。为此，我们提出了SALT，一个与模型无关的抽取式框架，它将每个句子的关键词组织成一个按句子频率（SF）排序的trie，这是一种轻量级、可复用的文档主题结构代理。这种基于trie的组织方式平滑了内存分配，并防止主导主题垄断预算。多锚点检索会激活在任何深度由查询关键词标记的trie节点，并且该trie在对话轮次之间持久存在，支持多轮使用而无需重新编码文档。通过保留文档主题，SALT减少了长上下文提示的预填充计算和内存成本，同时仍可与针对解码时延和内存的KV缓存方法组合使用。

## Abstract
As large language models (LLMs) process increasingly longer prompts, computation and KV-cache memory costs have emerged as major bottlenecks in inference systems. Existing input-level prompt compression methods address this, but rank each sentence by a scalar relevance score, treating the document as an unstructured pool of words and sentences. Under tight budgets, this causes theme collapse, where the dominant theme(s) of a document consumes the budget, discarding less-frequent yet task-relevant themes. Preserving thematic coverage instead requires allocating the budget across recurring themes rather than scoring sentences in isolation. To this end, we propose SALT, a model-agnostic extractive framework that organizes per-sentence keywords into a trie ordered by sentence frequency (SF), a lightweight, reusable proxy for document thematic structure. This trie-based organization smooths memory allocation and prevents dominant themes from monopolizing the budget. Multi-anchor retrieval activates trie nodes labeled by query keywords at any depth, and the trie persists across dialogue turns, supporting multi-turn use without re-encoding the document. By preserving document themes, SALT reduces the prefill computation and memory cost of long-context prompts while remaining composable with KV-cache methods that target decoding-time latency and memory.

### 2. 论文提出的方法论
SALT 是一种模型无关的抽取式框架，其核心是将提示压缩重新定义为在预算约束下**保持主题覆盖**的问题。它通过一个按句子频率（SF）排序的词汇Trie结构来实现，主要包含两个阶段。

- **索引阶段（Theming & Trie Construction）**:
  - **关键词提取**: 使用轻量级编码器（`BGE-small-en-v1.5`）为文档的每个句子提取关键词。具体利用`[CLS]` token 的注意力权重对内容词进行排序，并通过“膝部检测”算法，根据词嵌入重建整句含义的余弦相似度曲线，动态确定保留的关键词数量 $k_i$，得到关键词集合 $K_i$。
  - **计算句子频率**: 统计每个关键词在不同句子中被选为关键词的句子频率 $SF(w) = |\{i : w \in K_i\}|$。高句子频率意味着该词参与了文档中反复出现的词汇结构，是文档级的显著词。
  - **构建显著词汇Trie**: 将句子频率高于 $p$ 分位数（默认为0.9）的关键词作为显著词集 $S$。对于每个句子，将其关键词集 $T_i = K_i \cap S$ 按句子频率降序排列后，插入一个前缀树（Trie）。Trie 的内部节点为显著关键词，叶节点存储句子ID。共享相同高显著度关键词的句子共享Trie前缀，分支点揭示了主题的差异性。

- **选择阶段（Budget-Constrained Selection）**:
  - **主题预算分配**: 这是SALT的核心机制。在选择句子前，首先将压缩预算 $B$ 在Trie的活跃分支间进行分配。计算公式为固定配额加上与分支未覆盖关键词质量 $M_b$ 相关的残差份额，并使用次线性指数 $\alpha$，以确保低质量分支也能获得基础预算，从而**防止主题塌缩**。
  - **分支内句子评分与选择**: 在每个分支内部，根据句子对减少该分支 **未覆盖关键词质量** 的边际贡献 $\Delta_b(s_i | R)$ 进行评分，并考虑句子长度偏好和位置先验。句子评分公式为$score_b(s_i | R) = \Delta_b(s_i | R) \cdot L(n_i) \cdot I(s_i)$。这意味着重复已有主题锚点的句子价值会降低。
  - **查询模式**: 支持`summary mode`（无条件压缩）和`query mode`（查询相关压缩）。查询模式下，查询关键词可以激活Trie中任何深度的匹配节点（多锚点检索），从而引导预算向查询相关证据倾斜。
- **算法流程**: 整体流程为先进行锚点阶段（查询模式），再对活跃分支进行预算分配，之后执行分支内句子选择，最后进行全局填充以用尽预算，输出按原文顺序排列的句子子集。

### 7. 优点
- **理论基础扎实**: 将压缩问题从标量排序重新定义为“主题覆盖”问题，并明确提出了“主题塌缩”现象，提供了明确的动机和解决方案。
- **方法设计巧妙**: 提出的**句子频率（SF）排序的词汇Trie**是一种创新且轻量级的主题结构代理，巧妙地将NLP中的注意力机制和信息检索中的数据结构（Trie）相结合。
- **预算分配策略独特**: **先分配后选择**的机制是SALT的核心亮点，从结构上解决了主题塌缩问题，这与只做后处理惩罚的传统方法有本质区别。
- **实用性强**: SALT作为一个模型无关的预处理器，输出纯文本，能够无缝集成到任何LLM推理管线中，并能与解码端的KV缓存优化方法组合使用，且不依赖新型GPU硬件，具有极高的实用价值。
- **实验评估全面**: 从准确率、延迟、内存、硬件移植性到多轮对话成本，评估维度非常全面，有力地支撑了其结论。

### 8. 不足与局限
- **代码任务的性能**: 由于SALT以句子为粒度进行操作，难以捕捉代码对token和行的精确结构要求，导致其在代码任务上表现不佳，显著落后于全上下文基线和token级别的KV缓存方法。这是所有基于句子的预处理方法的通病。
- **压缩率的下限**: SALT的性能受其句子级抽取粒度的限制，存在一个压缩率下限（论文指出通常在5%-20%）。它无法像token级别的KV缓存方法那样实现更极致的压缩。
- **关键词提取器依赖**: 其主题构建依赖于一个预训练的编码器（BGE-small），该编码器的注意力模式质量直接影响关键词（尤其是对代码等特殊文本）的提取效果。虽然该方法轻量，但这仍引入了一个外部模型依赖。
- **主题覆盖的潜在偏差**: 使用词频作为显著性的唯一衡量标准，可能会忽视一些出现频率低但语义关键的主题词（如专有名词、特定术语），尽管这些词可能会在句子选择过程中以非关键词身份被保留。
